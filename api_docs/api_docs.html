Introduction

Welcome to the Room Chat API documentation. This API powers a real-time discussion platform allowing users to create rooms, chat, and manage friends.

Base URL: https://xfn8hgh7pt.ap-south-1.awsapprunner.com/api/

Authentication: Most endpoints require Token Authentication. Include the token in the header:

Authorization: Token <your_token>

Architecture & Hosting

The Room Chat API utilizes a modern, containerized architecture deployed on AWS. The system is designed for scalability and ease of deployment.

Compute: AWS App Runner

A fully managed service that provides a fast, simple, and secure way to deploy containerized web applications and APIs, at scale and with no prior infrastructure experience required. It handles autoscaling and load balancing automatically.

Containerization: Docker

The application is packaged into a lightweight, portable container using Docker. This ensures consistency across development and production environments. The image is built on top of python:3.12-slim.

Registry: AWS ECR

Amazon Elastic Container Registry (App Registry) is used to store, manage, and deploy the Docker container images. App Runner pulls the latest image from ECR for deployment.

Database: Neon DB

Serverless PostgreSQL database. The application connects to Neon via dj-database-url, allowing for scalable storage that separates compute from storage.

Web Server: Gunicorn

The Python WSGI HTTP Server acts as the interface between the Python application and the web. It is configured to run with multiple workers for handling concurrent requests.

Static Files: WhiteNoise

Middleware that allows the web app to serve its own static files (CSS, JS, Images), making it self-contained without needing a separate CDN setup for basic assets.

Authentication

Login

POST /users/login/

Authenticate a user and retrieve a token.

{
  "username": "myuser",
  "password": "mypassword"
}

Register

POST /users/register/

Register a new user account.

{
  "username": "newuser",
  "email": "user@example.com",
  "password": "securepassword"
}

User Operations

Get Profile

GET /users/profile/

Retrieve the currently logged-in user's profile information.

Update Profile

PUT /users/profile/

Update user details (username, email, password).

Search Users

GET /users/search/?q=<query>

Search for users by username. Supports partial and phonetic (Soundex) matching.

Friends & Direct Messages

List Friends

GET /users/friends/

Get a list of friends for the current user.

Add Friend

POST /users/friends/

Add a user as a friend.

{ "username": "target_user" }

Start Direct Chat

POST /chat/start/

Initiate a DM room with another user. Returns existing room if found.

{ "username": "target_user" }

Rooms

List Rooms

GET /rooms/?q=<query>

Get a list of public chat rooms. Supports filtering by topic, name, or description.

Create Room

POST /rooms/

Create a new public chat room.

{
  "topic_name": "Python",
  "name": "Django Developers",
  "description": "Discussing Django Rest Framework"
}

Get Room Details

GET /rooms/<id>/

Retrieve specific details of a room.

Join/Leave Room

POST /rooms/<id>/join/

Toggle participation in a room.

Messages

Get Room Messages

GET /rooms/<id>/messages/

Retrieve all messages for a specific room.

Send Message

POST /rooms/<id>/messages/

Post a new message to a room.

{ "body": "Hello world!" }

Utility

Get Topics

GET /topics/

Retrieve a list of all active discussion topics.

Recent Activity

GET /activity/

Get a stream of the most recent messages across all public rooms.
